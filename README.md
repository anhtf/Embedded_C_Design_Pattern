# Embedded_C_Design_Pattern
> Last updated: 2026-04-10

## Tổng quan
Repository này giới thiệu các mẫu thiết kế (Design Patterns) cho lập trình Embedded C. Mỗi thư mục chứa một ví dụ cụ thể, giải thích lý thuyết và cách áp dụng trong hệ thống nhúng.

## Nội dung chính
- **Introduction:** Giới thiệu chung về các pattern và nguyên lý thiết kế phần mềm nhúng.
- **Callback Pattern:** Sử dụng con trỏ hàm để xây dựng kiến trúc hướng sự kiện.
- **Bridge Pattern:** Tách biệt abstraction và implementation trong HAL.
- **Conditional Pattern:** Áp dụng cấu trúc điều kiện để điều khiển luồng logic.
- **Concurrency Pattern Overview:** Tổng quan về các pattern đồng bộ và đa luồng.
- **Mutex Pattern:** Bảo vệ tài nguyên dùng chung bằng mutex.
- **Semaphore Pattern:** Đồng bộ cạnh tranh sử dụng semaphore.
- **Spinlock Pattern:** Khóa vòng lặp cho hệ thống thời gian thực.
- **Singleton Pattern:** Quản lý tài nguyên duy nhất trong hệ thống.
- **Object Pattern:** Mô phỏng đối tượng trong C với encapsulation.
- **Opaque Pattern - Static Memory Pool:** Ẩn chi tiết triển khai và quản lý bộ nhớ tĩnh.
- **Return Value Pattern:** Xử lý lỗi và trả giá trị trạng thái.
- **Inheritance Pattern:** Mô phỏng kế thừa trong C.
- **Virtual API Pattern (Polymorphism):** Sử dụng bảng hàm để mô phỏng đa hình.
- **Factory Pattern:** Tạo đối tượng bằng factory để giảm coupling.

## Mục tiêu
- Học cách áp dụng các pattern thiết kế phổ biến trong môi trường nhúng.
- Giảm độ kết dính chặt và cải thiện khả năng mở rộng cho mã C.
- Thực hành với các mẫu thiết kế phù hợp cho hệ thống tài nguyên hạn chế.
